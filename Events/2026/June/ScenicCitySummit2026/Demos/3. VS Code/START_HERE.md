# 03 - VS Code Demo

## Goal

Show VS Code as a local data professional workbench:

files, CSV inspection, notebooks, SQL, Docker, terminal, and Git.

## Demo Flow

1\. Open this folder in VS Code

2\. Show folder structure

3\. Open `data/fallout\_vaults.csv`

4\. Enable Rainbow CSV

5\. Enable CSV Lint

6\. Open `notebooks/01\_local\_data\_check.ipynb`

7\. Run local notebook using the `.venv` kernel

8\. Show `uv` setup in terminal

9\. Show Docker SQL Server container

10\. Run SQL query from `sql/03\_demo\_queries.sql`

11\. Show Source Control briefly



\## Talking Point



VS Code is not just a code editor. For data work, it can become a local workbench where files, notebooks, SQL, terminals, Docker, and Git all meet.
