\# Docker / Container Tools Demo



\## Goal



Show that VS Code can manage the same local containers normally managed through Docker Desktop.



\## Demo Flow



1\. Open Container Tools in the VS Code sidebar

2\. Expand Containers

3\. Show the SQL Server container

4\. Start the container if it is stopped

5\. View logs briefly

6\. Switch to SQL Server extension

7\. Connect to `localhost,1433`

8\. Run `SELECT @@VERSION`



\## Talking Point



Docker Desktop is still the engine, but VS Code lets me manage the container from the same workspace where I have notebooks, SQL files, terminal, Git, and project notes.
